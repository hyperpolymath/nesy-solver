== Overview

This guide covers packaging, deploying, and maintaining nesy-solver for
distribution on your platform.

== Runtime Dependencies

Deno 2.x, ReScript 11.x

== Build from Source

[source,bash]
----
git clone https://github.com/hyperpolymath/nesy-solver.git
cd nesy-solver
just build-release
----

Output: `dist`

== Packaging

=== Guix

[source,bash]
----
guix build -f guix.scm
----

=== Nix

[source,bash]
----
nix build
----

=== Container (Stapeln)

[source,bash]
----
just stapeln-export    # Generates Containerfile
podman build -t nesy-solver .
----

=== Manual Package

[source,bash]
----
just install --prefix=/usr/local
----

Files installed:

[cols="1,2"]
|===
| Path | Contents

| `$PREFIX/bin/`
| Executables

| `$PREFIX/share/nesy-solver/`
| Data files, assets

| `$PREFIX/share/doc/nesy-solver/`
| Documentation

| `$PREFIX/share/applications/`
| .desktop file (Linux, if GUI)

| `$PREFIX/share/man/man1/`
| Man pages
|===

== Configuration

Default config location: `$XDG_CONFIG_HOME/nesy-solver/config.toml`

Fallback: `$HOME/.config/nesy-solver/config.toml`

== Health Checks

[source,bash]
----
just doctor            # Full diagnostic
just run --version   # Version check
just run --selftest  # Built-in self-test
----

== Updating

[source,bash]
----
git pull
just build-release
just install --prefix=/usr/local
----

Or via OPSM: `opsm update nesy-solver`

== Multi-Instance Deployment

For deploying multiple instances (e.g., different users or tenants):

[source,bash]
----
just install --prefix=/opt/nesy-solver-instance1 --config=/etc/nesy-solver/instance1.toml
just install --prefix=/opt/nesy-solver-instance2 --config=/etc/nesy-solver/instance2.toml
----

Each instance has isolated config, data, and logs.
